01 · El qué
Presenta: Oscar Morán

NUVIO: streaming ecuatoriano

  • Qué hace: licencia y cataloga contenido audiovisual, lo ofrece por suscripción.
  • Alcance: usuarios, ciudades y provincias de Ecuador.
  • Modelo: diagrama E/R conceptual completo.
01 · El qué
Presenta: Oscar Morán

Cuatro bloques del MER

  • Geografía: Región → Provincia → Ciudad.
  • Identidad: Persona es la base; User y Actor son especializaciones 1:1.
  • Catálogo: Estudio/Productora producen Película/Serie; Categoría clasifica N:M.
  • Consumo: User contrata Plan; Perfil visualiza y guarda listas.

Decisión clave: Empresa y Logs quedan fuera de las relaciones de negocio — son entidades administrativas (institucional / auditoría transversal), no operativas.

02 · El cómo
Presenta: Jim Vargas

Del E/R al esquema relacional

  • Toda entidad → tabla con id SERIAL (surrogate key).
  • 1:N → FK en la tabla del lado "muchos" (ej. pelicula.estudio_id).
  • N:M → tabla puente nueva (pelicula_categoria, reparto_pelicula).
02 · El cómo
Presenta: Jim Vargas

Normalización y nomenclatura

  • 1FN: atributos atómicos, sin grupos repetitivos.
  • 2FN: surrogate keys evitan dependencias parciales.
  • 3FN: sin dependencias transitivas — separar Persona/User/Actor evita duplicar cédula y correo si una persona es ambas cosas.
  • Nomenclatura: snake_case + bloque de auditoría estándar (is_active, created_at, updated_at, created_by, updated_by) en cada tabla.
03 · El script
Presenta: Matías Yépez

Reto: dependencia circular

persona.created_by → user.id, pero user.persona_id → persona.id.

ddl_persona_user.sql
-- 1. persona sin FK de auditoría · 2. user · 3. cerrar el ciclo:
ALTER TABLE persona ADD CONSTRAINT fk_persona_created_by
  FOREIGN KEY (created_by) REFERENCES "user"(id);
03 · El script
Presenta: Matías Yépez

Reto: patrón polimórfico

Sin tabla genérica "contenido" para referenciar película o capítulo.

ddl_visualizacion.sql
CREATE TABLE visualizacion (
  ...
  pelicula_id INT REFERENCES pelicula(id),
  capitulo_id INT REFERENCES capitulo(id),
  CHECK (pelicula_id IS NOT NULL OR capitulo_id IS NOT NULL)
);
